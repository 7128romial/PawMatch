# פסאודו-קוד מתוקן: מסלול הפעולה המלא מקצה לקצה — PawMatch

---

## 0.0 התחלה
הסוכן מציג עצמו בקצרה ועובר לשאלה הסגורה הראשונה.

---

## 1.0 בדיקות קלט (מתבצעות על כל הודעה, לפי הסדר)

**1.1** אם ההודעה ארוכה מ-500 תווים:
→ החזר שגיאה ובקש לקצר.

**1.2** אם זוהה מידע אישי רגיש — בדיקה **דטרמיניסטית** ל-9-10 ספרות ברציפות (ת"ז / טלפון):
→ אל תאחסן, הצג הודעת אזהרה וחזור לשאלה האחרונה.
*(שאר ה-PII — מידע רפואי, צבאי, כתובת — מטופל על ידי ה-LLM ב-system prompt)*

**1.3** אם זוהתה כוונה להזיק לכלב — בדיקה **דטרמיניסטית** על patterns ספציפיים (אגרסיה, קרבות, הזנחה, רבייה מסחרית):
→ עצור את תהליך ההתאמה לחלוטין, והסבר את הסיבה האתית.

**1.4** אם הקלט הוא גיבריש (keyboard mash / סמלים בלבד) — בדיקה **דטרמיניסטית**:
→ בקש מהמשתמש לנסח מחדש, אל תעביר ל-NLP.

**1.5** אם הקלט אינו קשור לאימוץ (חתולים, מזג אוויר, רפואה) — מזוהה על ידי ה-LLM:
→ החזר את המשתמש לנושא בעדינות.

## 2.0 פילטרים קשים (דרישות פיזיות מוחלטות)

**2.1** קליטת 4 פרמטרים קשיחים מהמשתמש **בסדר הבא** (לכל אחת אפשרות "אין לי העדפה"):
1. **גודל** (קטגוריה: קטן / בינוני / גדול) ← קטגוריה, לא משקל בק"ג
2. **גיל** (קטגוריה: גור / בוגר / מבוגר)
3. **מין** (זכר / נקבה)
4. **צבע** (קטגוריה: שחור / לבן / חום / אפור / מעורב)

**2.2** סנן את כלל הכלבים במאגר לפי הבחירות.

**2.3** אם לא נשאר אף כלב במאגר שעונה על הדרישות הפיזיות:
→ עצור, הצג הודעה המציעה להרפות פילטרים פיזיים, ואל תמשיך לחישוב מתמטי על קבוצה ריקה.

## 3.0 חילוץ פרופיל באמצעות ה-LLM (רק אחרי הפילטרים)

**3.1** הזמן את המשתמש לתאר את אורח חייו בשפה חופשית.
- תמיכה מובנית של gpt-4o בעברית ובאנגלית — **ללא שלב תרגום**, המודל מחלץ פרמטרים ישירות מהקלט בכל שפה שהיא.
- הטקסט הגולמי נצבר לאורך כל השיחה לשימוש בשלב הנימוקים (שלב 8).

**3.1.5** Safety Net דטרמיניסטי (מנגנון גיבוי לאחר תשובת ה-LLM):
- אם הפרמטר הפעיל לא חולץ על ידי ה-LLM, אך המשתמש נתן תשובה ברורה (למשל "אני בעבודה" לשאלת שעות לבד) — מנוע דטרמיניסטי ממלא את הערך ישירות (1 או 5) ומדלג על ה-acknowledgment של ה-LLM.

**3.2** חילוץ פרמטרים לפי שלוש רמות שאלות:

### 3.2.1 שאלה חיונית (Tier A — חובה)
מאפיינים: `a1_adapts_well_to_apartment_living`, `e3_exercise_needs`, `a4_tolerates_being_alone`, `d5_tendency_to_bark_or_howl`

**בפעם הראשונה (retry_count = 0):**
- מידע ברור שהוזכר → חלץ והקצה ערך מספרי קשיח (1 עד 5).
- מידע לא ברור / התחמקות → שאל שנית בניסוח שונה לחלוטין, **אל תקצה ערך עדיין**.

**בפעם השנייה (retry_count ≥ 1):**
- מידע ברור שהוזכר → חלץ והקצה ערך מספרי קשיח (1 עד 5).
- מידע חלקי או מאפיין שלא הוזכר כלל → הקצה ערך ניטרלי קבוע (3).

### 3.2.2 שאלה מותנה (Tier B — מותנה)
מאפיינים: `b3_dog_friendly`, `b2_incredibly_kid_friendly_dogs`, `d1_easy_to_train`, `c1_amount_of_shedding`, `a2_good_for_novice_owners`

- מידע ברור שהוזכר → חלץ והקצה ערך מספרי קשיח (1 עד 5).
- מידע חלקי או מאפיין שלא הוזכר כלל → הקצה ערך ניטרלי קבוע (3) בחישוב הסופי (המשקלים **אינם** מחושבים מחדש).

### 3.2.3 שאלה משנית (Tier C — פסיבי)
מאפיין: `c2_drooling_potential`

- **אל תשאל עליו ביוזמת המערכת בשום שלב.**
- אם המידע הוזכר מיוזמת המשתמש → חלץ והקצה ערך מספרי קשיח (1 עד 5).
- מידע חלקי או מאפיין שלא הוזכר כלל → הקצה ערך ניטרלי קבוע (3).

## 5.0 חישוב התאמה (Weighted Cosine Similarity)

**5.1** בנה את וקטור המשתמש — ערך ברירת מחדל 3 לכל תכונה שלא עלתה בשיחה.

**5.2** נרמל את וקטור המשקלים: חלק כל משקל בסכום כל המשקלים כך שסכומם יהיה 1.0.

**5.3** עבור כל כלב שעבר את הפילטרים:
- **5.3.1** הכפל כל ערך בשורש המשקל **המנורמל** שלו (`sqrt(normalized_weight)`).
- **5.3.2** חשב Cosine Similarity מול וקטור המשתמש המשוקלל ושמור את הציון.

**5.4** מיין את הכלבים מהציון הגבוה לנמוך.

**5.5** החלט לפי סף ההתאמה:
- **5.5.1** אם אף כלב אינו מעל 75% → הצע להרחיב את הפילטרים.
- **5.5.2** בחר את שלושת הכלבים המובילים (Top 3).

## 7.0 טיפול בבקשת גזע ספציפי

**7.1** אם הגזע נמצא אחרי הסינון → הצג אותו בראש הרשימה.

**7.2** אם הגזע אינו זמין (לא עבר את הפילטרים הפיזיים):
- **7.2.1** קח את **הווקטור ההתנהגותי** של הגזע המבוקש כוקטור חיפוש.
- **7.2.2** מצא 3 כלבים דומים **מכלל הכלבים שעברו את הפילטרים הפיזיים** (ללא הגבלה לקלאסטר ספציפי) לפי Cosine Similarity.
- **7.2.3** הצג אותם כחלופה מנומקת — הסבר שהגזע המבוקש לא זמין אך אלה כלבים בעלי פרופיל התנהגותי דומה.
